Signing Requests

Description about singing requests..

Users

Get Running Stacks

Get user's total running stacks

GET /user/{uid}/get/stacks

Example response:

{
    "stacks":"2"
}

Get Running Nodes

Get user's total running server nodes

GET /user/{uid}/get/nodes

Example response:

{
    "nodes":"6"
}

Get Metrics

Get user's total running stacks, nodes & metrics

GET /user/{uid}/get/metrics

Example response:

{
    "stacks":"2",
    "nodes":"6",
    "cpuavg":"0.11",        //average of all cpus' load in %
    "latency":"0.0000",     //seconds
    "maxnetworkin":"3",     //bytes
    "maxnetworkout":"3",    //bytes
    "sumrequests":"0",
    "statuscheckfailed":"0"
}

Get Stack Status

Get user's stack status

GET /user/{uid}/get/stackstatus?stack_ids={id1},{id2} HTTP/1.1

Example response:

[
    {
        "mo-54477f1b43535-d3ubs3r7x":"DELETE_COMPLETE"
    },
    {
        "mo-54477f1b43535-uo7enam02":"CREATE_COMPLETE"
    }
]

Deployment

Create Stack

Create a new stack

POST /deploy/{uid}/{region}/stack/start?configurations={...}

Example parameters:

{
    "region":"ap-northeast-1",    // see full list of available regions
    "architecture":"art_elb",     // or "art_single"
    "os":"os_centos",
    "software":[
                  "sw_nginx",     // see full list of available software configurations
                  "sw_ruby"
                ],
    "database":[
                  "db_redis",     // see full list of available database configurations
                  "db_mongodb"
                ]
}

Example response:

{
    "stack_name":"mo-54477f1b43535-s07cav0w4",
    "status":"CREATE_IN_PROGRESS"
}

Details:

Parameter architecture has two values: art_single and art_elb.

If you create a single structured stack there will be only one server node deployed and it cannot be scaled up or down.

If you create a art_elb structured stack means you create server node(s) within a VPC with several subnets and with auto-scaling groups. You are able to scale up or down based on time or traffic triggers. You can specify the min and max number of server nodes.

Update Stack

Update an existing stack

POST /deploy/{uid}/{region}/stack/update

Save Stack

Save a stack's configuration

POST /deploy/{uid}/{region}/stack/save

Delete Stack

Delete an existing stack

POST /deploy/{uid}/{region}/stack/delete

Describe

Describe Stacks

Describe stack detailed information. If stack_ids is not passed, API will return all stacks information.

GET /describe/{uid}/stack

Example response:

{
    "stacks":"2"
}
